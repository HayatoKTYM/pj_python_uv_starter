## プロジェクト構成

```
project_root/
├── app/                    # アプリケーションコード
│   ├── __init__.py

│   ├── api/               # API関連
│   ├── models/            # データモデル
│   ├── services/          # ビジネスロジック
│   └── utils/             # ユーティリティ
├── tests/                 # テストコード
│   ├── __init__.py

│   └── test_*.py         # テストファイル
├── docs/                  # ドキュメント
├── scripts/               # スクリプト
├── .github/               # GitHub設定
│   └── workflows/         # GitHub Actions

├── uv.lock                # 依存関係ロック

```

### 命名規則

- **パッケージ/モジュール**: `lowercase_with_underscores`
- **クラス**: `PascalCase`
- **関数/変数**: `snake_case`
- **定数**: `UPPER_CASE_WITH_UNDERSCORES`
- **プライベート**: `_leading_underscore`

## コーディング規約

### フォーマッターとリンター

このプロジェクトでは **Ruff** を使用します：

```toml
[tool.ruff]
line-length = 100
target-version = "py311"

[tool.ruff.lint]
select = [
    "E",   # pycodestyle errors
    "W",   # pycodestyle warnings
    "F",   # pyflakes
    "I",   # isort
    "B",   # flake8-bugbear
    "C4",  # flake8-comprehensions
    "UP",  # pyupgrade
]
```

### コミット前の確認

```bash
# フォーマット
uv run ruff format .

# リントチェック
uv run ruff check --fix .

# 型チェック
uv run mypy app

# テスト
uv run pytest
```

## 依存関係管理

### uv の使用

```bash
# 依存関係の追加
uv add <package-name>

# 開発依存関係の追加
uv add --dev <package-name>

# 依存関係の更新
uv lock --upgrade

# 環境の同期
uv sync
```

### バージョン指定

- **本番依存関係**: できるだけ緩い制約（例: `fastapi>=0.109.0`）
- **開発依存関係**: 最新版を使用
- **重要な依存関係**: 互換性が重要な場合は厳格に指定

## テスト

### テストの原則

1. **AAA パターン**: Arrange（準備）、Act（実行）、Assert（検証）
2. **1テスト1検証**: 各テストは1つの概念を検証
3. **独立性**: テストは互いに独立して実行可能
4. **高速性**: 単体テストは高速に実行

### テストファイル構成

```python
"""Test module for items functionality."""
import pytest
from fastapi.testclient import TestClient


def test_create_item() -> None:
    """Test item creation."""
    # Arrange
    client = TestClient(app)
    item_data = {"name": "Test"}
    
    # Act
    response = client.post("/items", json=item_data)
    
    # Assert
    assert response.status_code == 201
    assert response.json()["name"] == "Test"
```

### カバレッジ目標

- **最低ライン**: 80%
- **推奨**: 90%以上
- **重要な機能**: 100%

    ...

## Git運用

### ブランチ戦略

- `main`: 本番リリース用
- `develop`: 開発統合用
- `feature/*`: 機能開発用
- `fix/*`: バグ修正用
- `hotfix/*`: 緊急修正用

### コミットメッセージ

Conventional Commits 形式を推奨：

```
<type>: <description>

[optional body]

[optional footer]
```

**Types:**
- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメント
- `style`: フォーマット
- `refactor`: リファクタリング
- `test`: テスト
- `chore`: その他

**例:**
```
feat: add user authentication endpoint

Implement JWT-based authentication with refresh tokens.

Closes #123
```

## CI/CD

### CI パイプライン

すべてのPRで以下を実行：

1. **フォーマットチェック**: `ruff format --check`
2. **リントチェック**: `ruff check`
3. **型チェック**: `mypy`
4. **テスト**: `pytest --cov`
5. **カバレッジレポート**: Codecovへアップロード

### リリースプロセス

1. `develop` ブランチでの開発・テスト
2. PRレビュー
3. `main` へのマージ
4. バージョンタグの作成
5. 自動デプロイ（設定がある場合）
